Prefix Expressions

Note: All code should be written within this template directory. Test cases have been provided in the folder tests, and can be run using make test.

In this assignment, you will evaluate expression in prefix notation, which is particularly amenable to a recursive solution. An expression in prefix notation will be of the form "operator argument argument", for example "+ 1 2" evaluates to three. If you wish to evaluate "+ * 6 7 - 10 2", you would first reduce the expression to "+ 42 8", and then to 50.

To help you, a function to parse the input has been provided (read_token), along with an example of how it can be used (print_expression). Each call to the function read_token will return the next number or operator to you.

Sample Input/Output

Input: / - * + 3 4 7 3 5
Output: 9
